7 # What Is It?
8 
9 Segnix is an open-source interface library for hardware manipulation
10 based on Linux platform and compatible with [Arduino API][1].
11 You can develop with it by using C/C++ and Python. Segnix is dedicated to provide
12 a fast, efficient and unified software development environment for the person
13 like you with a strong electronic design idea.
14 
15 # CAUTION!
16 
17 ## Before November 22, 2014, Segnix was named ITEAD-OS-SDK
18 
19 # What Can It Do?
20 
21 Through Segnix, users can directly access to the Linux platform hardware
22 resources such as GPIO, UART, SPI, I2C etc. Segnix supports various
23 hardware platforms, such as Iteaduino Plus A10/A20, IBOX,
24 ITEAD Core EVB, Raspberry Pi and Raspberry Pi Model B+ and so on. Meanwhile, It also
25 provides a great amount of libraries for all kinds of child board like NFC(PN532)
26 module, GPS shield, GSM(SIM900/SIM908) shield, nRF24L01 module, OLED and LCD module etc.
27 
28 Simplifying the hardware operation makes it possible for software engineers
29 who are not familiar with the underlying hardware and junior developers to
30 understand more about "Code World" and "Physical World". It can also work
31 as a rapid prototyping development tool.
32 
33 # Features
34 
35 At present, Segnix has a set of **core features** as below:
36 
37  - Support for upper-layer time functions
38  - Support for PWM output with any GPIO
39  - Support for GPIO operations
40  - Support for GPIO 8bit/16bit bus operations
41  - Support for UART bus operations
42  - Support for I2C bus operations
43  - Support for SPI bus operations
44  - Support for external interrupts operations(Only for `ITEAD CORE EVB`)
45  - Support for ADC (Only for `BeagleBone Black`)
46 
47 Based on core features, plenty of **functional features** can be reached:
48 
49  - Support for SSD1306-based I2C OLED screen operations
50  - Support for 1602 LiquidCrystal module
51  - Support for GPS module
52  - Support for GSM(SIM900/SIM908) module including GPRS/GPS operations
53  - Support for NFC(PN532) module
54  - Support for nRF24L01 module
55  - Support for Temperature and Humidity module
56  - Support for [IoTgo platform](http://iotgo.iteadstudio.com)

196 ## Compilation Helper: segnix
197 
198 segnix is a script to compile your program conveniently. before November 22, 2014,
199 segnix was named iteadcompile. Actually, segnix is a link to iteadcompile.

204  segnix objname source.ino [source2.ino] [...]
205  segnix objname source.cpp [source2.cpp] [...]
206  segnix objname source.c [source2.c] [...]
207  Examples for .ino
208  segnix hello hello.ino
209  segnix led led1.ino led2.ino
210  segnix led *.ino
211  Examples for .cpp
212  segnix hello hello.cpp
213  segnix led led1.cpp led2.cpp
214  segnix led *.cpp
215  Examples for .c
216  segnix hello hello.c
217  segnix led led1.c led2.c
218  segnix led *.c
219  NOTES:
220  objname cannot cantian ".ino" or ".c" or ".cpp"
221  Version:
222  segnix [-v]
223  segnix [--version]

225 ## To use Segnix in Python
226 
227 Create a new file named `blink.py` which reads as follow:
228 
229  #! /usr/bin/env python
230  from iteadsdk import *
231  LED = PG9
232  pinMode(LED, OUTPUT)
233  while True:
234  digitalWrite(LED, HIGH)
235  delay(1000)
236  digitalWrite(LED, LOW)
237  delay(1000)
238 
239 Make `blink.py` executable:
240 
241  $ chmod +x blink.py
242 
243 Then, run it:
244 
245  $ sudo ./blink.py
246 
247 You can get commutative HIGH and LOW from GPIO PG9, if you've run it on ITEAD CORE EVB
248 or other platforms based on Allwinner A10/A20 SoC.

358 #### Note: How to understand debug information from compiler
359 
360 You DO NOT include the itead.h explicitly in your source file with suffix .ino.
361 segnix will take care of everything. That means you need to know the method
362 you should use for locating your .ino sketch bugs. Actually,if you have a sketch
363 named "Hello.ino" with a bug at level ERROR in line 25, you will get some ERROR
364 informations telling the ERROR bug in "Hello.cpp" with line 26(25+1) when
365 command "segnix Hello Hello.ino" is executed. Why? segnix insert one
366 line like `include "itead.h"` in the head of your Hello.ino, change the suffix
367 .ino to .cpp and compile(using g++) Hello.cpp to generate Hello which is
368 executable binary file. When you encounter other bugs in your sketch,debug
369 skills on C++ program will also helpful for solving your problems.
